## 1. 업무 개요

- **주제**: Pragmatis Arcade Hub HUD 표준화 및 UI 중복 문제 해결

## 2. 수립된 프로젝트 표준 규칙 (Arcade HUD Standard)

### 2.1. 3분할 HUD 레이아웃 구조
향후 추가되는 모든 아케이드 게임은 다음의 3분할 구조를 HTML/CSS로 구현해야 함.
- **[Left Section]**: 게임 핵심 정보 (점수, 코인 등)
- **[Center Section]**: 컨트롤 시스템 (일시정지, 로비 버튼)
- **[Right Section]**: 추가 상태 정보 (라이프, 월드명, 남은 시간 등)

### 2.2. 시스템 메뉴 버튼 규격 (Control Pills)
- **폰트**: `Orbitron` (Google Fonts)
- **형태**: Pill-shape (border-radius: 20px)
- **효과**: Hover 시 `scale(1.1)`, `filter: brightness(1.2)`, `0.3s` 트랜지션 적용.
- **색상 표준**:
    - **PAUSE**: `linear-gradient(135deg, #3b82f6 0%, #1d4ed8 100%)` (네온 블루)
    - **LOBBY**: `linear-gradient(135deg, #ec4899 0%, #be185d 100%)` (네온 핑크)

### 2.3. 포털-게임 통신 규약
- 게임 내 '메인화면으로' 버튼 클릭 시 반드시 다음 메시지를 부모 창에 전송해야 함.
  ```javascript
  window.parent.postMessage('backToLobby', '*');
  ```

### 2.4. 포털 UI 보호 로직
- 포털(`PragmatisGames.html`)은 외부 iframe 게임이 활성화될 때 자체 HUD를 `display: none`으로 전환하여 중복 출력을 방지함.
